## 0x00: 📝 项目介绍
**#Rigorous** 是一个基于 **C++11** 标准开发的轻量级机器学习静态库。该项目致力于提供简洁、高效且易于集成的算法实现，完全基于标准模板库（STL）开发，不依赖任何外部大型数学库（如 Eigen 或 OpenCV）。 🛠️

本项目不仅包含经典的机器学习算法（如 BP 神经网络、核 SVM），还内置了一个完整的**反向模式自动微分（Autograd）**系统，非常适合教学研究、嵌入式开发或轻量级 AI 逻辑集成。 🎓

## 0x01: ⚙️ 构建途径
由于本项目采用 **Header-only**（仅头文件）设计，您无需复杂的编译过程，只需将源代码集成到您的项目中即可。 📦

### 获取代码
```powershell
git clone "https://www.github.com/classzheng/rigorous.git"
```

### 集成方式
1. 将 `src/` 目录下的所有文件复制到您的工程目录或 Include 路径下。 📂
2. 在代码中包含主入口头文件：
   ```cpp
   #include "rigorous.h"
   ```
3. 确保编译器开启了 C++11 或更高版本的支持： 🛡️
   - GCC/Clang: `-std=c++11`
   - MSVC: `/std:c++11`

## 0x02: 🌟 功能概括
*   **🧠 Autograd (自动微分)**：支持反向模式自动微分，提供动态内存池管理运算节点。
*   **🕸️ NeuralNetwork (神经网络)**：支持多层隐藏层的 BP-ANN 实现，包含常见激活函数及其导数。
*   **⚖️ C-SVM (支持向量机)**：
    *   **BinarySVM**：基于 SMO 算法的二分类支持向量机。
    *   **MultipleSVM**：支持多分类任务。
    *   **LSSVM**：最小二乘支持向量机，通过线性方程组快速求解。
    *   **Kernel Trick**：内置 7 种核函数（线性、高斯、拉普拉斯、柯西、余弦、多项式、三角）。
*   **🌲 RandomForest (随机森林)**：*(🚧 开发中...)*。

## 0x03: 🔍 定义的函数与对象等可调用部分
所有功能均封装在 `Rigorous` 命名空间下。

### 1. 自动微分系统 (`namespace BackwardGrad`)
*   `Autograd<T>`: 核心数据类型，包含 `.data` (数值), `.grad` (梯度)。
*   `Automem<T>`: 自动内存管理池。 ♻️
*   **常用函数**: `AgSin`, `AgCos`, `AgTanh`, 以及重载的算术运算符。

### 2. 神经网络 (`namespace NeuralNetwork`)
*   `UnitLayer<T>`: 单层神经元定义。
*   `NetworkUnion<T>`: 神经网络整体容器。
    *   `forward(input)`: 前向传播。 ▶️
    *   `backward(target)`: 反向传播更新权重。 ◀️
    *   `train(data_pair)`: 批量训练。 🚄

### 3. 支持向量机 (`namespace SVMPackage`)
*   `Kernel<T>`: 核函数配置对象。 🧪
*   `BinarySVM<T>` / `MultipleSVM<T>`:
    *   `train(vx, ty, expects, limit)`: 训练模型。
    *   `predict(x)` / `decision(x)`: 进行预测或输出判定值。 🔮
*   `LSSVM<T>`: `optimize()` 快速优化。 ⚡

## 0x04: ⚠️ 注意事项
*   **内存管理**：在使用 `Autograd` 系统时，所有运算产生的临时节点均由 `Automem` 托管，请确保 `Automem` 对象的生命周期覆盖您的运算过程。 🧠
*   **模块开关**：如果为了提升编译速度，您可以在包含 `rigorous.h` 之前定义以下宏来禁用特定模块： 🔧
    *   `#define $DISABLE_ANN` (禁用神经网络)
    *   `#define $DISABLE_SVM` (禁用支持向量机)
    *   `#define $DISABLE_RF` (禁用随机森林)
*   **性能限制**：由于采用 STL 容器且未进行指令集优化（如 AVX/SSE），本项目在处理超大规模数据集时性能可能有限。 🐢
